# 筋トレ記録アプリ
個人利用の筋トレ記録アプリの開発仕様をまとめる。

## 技術選定
- まずはAndroidアプリとして動作すること（Pixel10）
- 公開しないので、開発ビルドのみで問題ない
- 将来的なマルチプラットフォーム展開を考慮してReactNativeを採用
- AIはGeminiを使用すること
- DBはSQLiteを使用すること

## 機能要件
### トレーニング記録
#### 記録項目
- 年月日
- トレーニング種目(Enum)
- 重量(kg)
- 回数
- セット数
- 備考
#### 記録方法
- 音声入力結果をAIで解析し、以下のフォーマットで出力する
    ```
    [
        {
            "date": "2024-06-01",
            "exercise": "ベンチプレス",
            "weight": 60,
            "reps": 10,
            "sets": 3,
            "notes": "フォームを意識して"
        },
        ...
    ]
    ```
- 日付は明示しない場合は当日の日付とする
- トレーニング種目は表記の揺れがあるため、アプリ内で定義したEnumにマッピングする（例：ベンチ→ベンチプレス）
- 一度に複数重量が指定された場合などは、複数の記録として保存する（例：ベンチ　30kg 10x3 20kg 10x1 -> 2データに分割）
- notes以外は必須項目とする

### トレーニング履歴
#### 閲覧
- カレンダー形式でトレーニング記録を閲覧できる
- トレーニング種目や期間でフィルタリングできる

#### 履歴操作
- 履歴の編集・削除ができる
- 編集は記録方法と同じく音声入力で行うこともできる

### 設定画面
- トレーニング種目の追加・編集・削除
    - 名前重複は許可しない
    - プリセットは以下の通り
        - インクラインダンベルベンチプレス
        - インクラインダンベルフライ
        - バーベルフルスクワット
        - ダンベルショルダープレス
        - デクラインシットアップ
        - インクラインダンベルベントオーバーロウ
        - ワンハンドダンベルロウ
        - 懸垂
### モデリング案
- historiesテーブル
    - id: int
    - date: date, index
    - exercise_id: int, index
    - weight: decimal
    - reps: int
    - sets: int
    - notes: string

- exercisesテーブル
    - id: int
    - name: string
